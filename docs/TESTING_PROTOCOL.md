# Testing Protocol

## Required Commands
- `npm run fixtures:supported`
- `npm run lint`
- `npm run typecheck`
- `npm test`
- `npm run build`
- `npm run smoke:electron`
- `npm run package:mac`

## Fixture Corpus
- Canonical support fixtures live in `test/fixtures/supported/`.
- The committed corpus is generated by `scripts/generate-supported-fixtures.mjs`.
- `test/fixtures/supported/manifest.json` is the source of truth for automated support coverage.
- The manifest declares:
  - fixture ids and source paths
  - single, compare, and folder scenarios
  - expected routed surface
  - expected selected assets
  - expected metadata
  - expected output excerpt
  - expected viewport kind and screenshot targets

## Automated Coverage
### Core
- every declared supported extension must have at least one manifest-backed fixture
- fixture family detection
- metadata extraction for text, image, gif, video, and unsupported controls
- routing decisions for single, compare, and folder intake
- compare-selection validation and manifest path existence

### Renderer
- single-view rendering for representative text, image, gif, and video entries
- text/code output renders as source content, not executed HTML or rich text
- compare rendering for text diff, image compare, gif compare, and video compare
- metadata panel content and unsupported-state rendering
- long-document viewport behavior and non-empty body output

### Built Smoke
- smoke runs against the compiled Forge bundle and the committed support corpus
- smoke loads scenarios from `test/fixtures/supported/manifest.json`
- smoke validates three layers for each covered scenario:
  - input analysis
  - output analysis
  - viewport analysis
- smoke uses a test-only debug bridge gated by `PRESENTER_TEST_MODE` to assert:
  - current session
  - selected asset ids and names
  - routed surface
  - compare capability
  - intake warnings
- representative screenshots are written to `output/playwright/`
- standard screenshot set includes:
  - plain text single view
  - text diff
  - image compare or diff
  - video compare
  - mixed grid
  - unsupported state

## Support Contract Rules
- adding a new supported extension requires adding committed fixtures and manifest entries in the same change
- unsupported files must remain visible as unsupported cards instead of disappearing silently
- text and code surfaces must display raw source content in the main viewport
- metadata must remain visible separately from the main content body
- the smoke suite must include at least one plain `.txt` single-view scenario

## Manual QA
- drag two image files onto the menubar icon and confirm direct 2-up open
- drag a folder and confirm grid browser routing
- verify closing the presenter window hides it without quitting the tray app
- verify the menubar app has no Dock icon

## Handoff Expectations
- include whether `npm run fixtures:supported` changed committed fixtures
- include whether the manifest-driven smoke run completed cleanly
- include the packaged `.app` and `.zip` paths
- include fresh screenshots from the changed renderer surfaces
- note any manual-only checks that were not fully automatable
